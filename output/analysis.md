## 根因 / Root Cause
通过对比项目源码 (`src/lib/concept-to-template.ts` 路由映射表及 `public/templates/` 目录树) 与初高中《物理》、《化学》、《生物》课程标准，发现当前项目的实验模板库存在明显的“偏科”与“不完备”现象。
1. **物理学科**：虽然涉猎广泛（已配置浮力、电路、密度、压强、运动学等基础模板），但在光学、复杂电磁学以及高中核心力学（如碰撞、平抛）上缺乏实际可运行的 HTML 模板。
2. **化学学科**：聚焦于反应速率、酸碱滴定、电解水等经典定量/定性实验，但在初中入门核心操作（如气体制备装置搭建）和有机化学微观结构上缺失。
3. **生物学科**：目前**完全处于空白状态**，缺乏显微视角交互和宏观生理过程模拟的基础基建。
4. **渲染瓶颈**：现有的渲染方式主要依赖原生 HTML DOM 和简单 Canvas，导致在处理需要强交互和动态演化计算的场景（如高中的刚体物理碰撞、复杂的有机微观结构 3D 展示）时表现吃力，缺乏专业引擎的辅助。

## 修改范围 / Scope
本项目当前为一个研究型分析任务，不需要修改现有代码，而是为未来阶段（Phase 3 及以后）的模板扩展提供指引图谱。以下是具体缺失的、高优先级的知识点模板清单：

### 一、 物理 (Physics) 缺失模板
**【初中高频/强交互需求】**
*   **光学综合（透镜成像与折射）**：目前缺失实际的交互模板。学生极需能够自由拖动物距、调节透镜焦距，实时观察“倒立放大/缩小”、“实像虚像”转化临界点的沙盒。
*   **摩擦力探究（控制变量法）**：通过切换不同粗糙表面的木板、增减砝码，动态读取弹簧测力计的示数。
*   **杠杆与滑轮组**：虽然概念有配置，但缺渲染层。需要能挂载不同位置的钩码，直观体验“省力不省距离”的原理。
**【高中难点/抽象概念】**
*   **动量守恒（碰撞实验台）**：完全弹性碰撞与非弹性碰撞的动态推演，通过调整小球质量和初速度，实时生成 v-t 图像。
*   **电磁感应综合（楞次定律）**：条形磁铁插入/拔出线圈，观察检流计指针偏转方向与磁通量变化的动态关系（极适合用可视化展示）。
*   **平抛/圆周运动**：调节初速度和抛射角，可视化运动轨迹及其水平/竖直分运动。

### 二、 化学 (Chemistry) 缺失模板
**【初中高频/仪器操作】**
*   **气体制备与装置气密性**：实验室制取氧气 (O₂) / 二氧化碳 (CO₂) 的装置拼装（发生装置与收集装置的匹配），以及加热前检查气密性的操作互动。
*   **质量守恒定律**：密闭系统内的白磷燃烧，动态展示微观原子的重新组合但不灭失的过程。
**【高中难点/微观与电化】**
*   **原电池与电解池对比**：铜锌原电池的电子流向、盐桥离子迁移微观动画（目前只有电解水，缺乏原电池）。
*   **焰色反应**：不同金属离子在火焰下的颜色呈现（纯视觉冲击力强，现实中很难看全所有颜色）。
*   **有机化学立体结构**：甲烷、乙烯、苯、乙酸乙酯的球棍模型三维旋转及取代/加成反应微观断键模拟。

### 三、 生物 (Biology) 缺失模板（目前完全空白，亟待建设）
**【微观操作与观察】**
*   **显微镜模拟器**：极高频需求。模拟调节粗/细准焦螺旋、转换物镜（低倍到高倍的视野亮度与范围变化）、移动玻片标本（倒立成像的上下左右移动）。
*   **洋葱表皮/口腔上皮细胞制作**：擦、滴、撕、展、盖、染的互动流程防错验证。
**【生理动态过程】**
*   **光合作用与呼吸作用**：探究光照强度、CO2 浓度对光合速率的影响（如产生气泡数计算），以及酵母菌有氧/无氧呼吸产物探究。
*   **质壁分离及复原**：调节外界溶液浓度，动态观察植物细胞液泡的失水萎缩与吸水膨胀过程。
*   **细胞分裂**：有丝分裂与减数分裂各时期染色体行为（联会、分离、赤道板排布）的慢镜头动画推演。
**【遗传学规律】**
*   **孟德尔遗传杂交模拟**：豌豆杂交实验的子代性状概率统计，支持动态设定基因型进行抽样模拟。

## 下游消费影响 / Downstream Impacts
*   **前端路由层**：新增实验需要在 `src/lib/concept-to-template.ts` 及 `template-registry.ts` 中注册对应的学科前缀（如 `biology/microscope`）。
*   **大语言模型层**：新增模板将极大地丰富 AI 能够推演的物理、化学及生物场景，使 Agent 能够处理更广泛的学生提问。
*   **UI/公共组件**：生物类宏观组织/微观细胞的渲染可能需要引入新的 2D 画布库或 SVG 动画资源，扩展现有的 `ui-core.css`。

## 风险评估 / Risks
*   **[P1] 生物学领域的架构拓展风险**：当前系统的渲染引擎主要为物理和化学（粒子、力学、反应速率等数值公式计算）设计。生物学高度依赖显微图像、时间序列动态演化及多节点微观过程，现有 HTML 方案可能在渲染性能和图元储备上面临挑战。
*   **[P2] 模板开发工作量过大**：物理的高中力学碰撞、化学的有机三维模型等开发成本较高，若使用纯原生 DOM+CSS，可能存在维护成本。建议评估是否引入外部轻量级引擎（如 Matter.js 用于力学碰撞，或 Three.js 简化版用于化学分子）。
