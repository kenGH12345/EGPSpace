## 何时使用

- 用户要求设计实验（任何学科）
- 用户输入概念并要求设计探究活动
- 需要为实验配置教学设计要素（变量分析、常见错误、观察点等）
- 跨学科项目式学习（PBL）设计

## 核心设计理念

### 自主推理 vs 规则灌输

❌ **旧模式（规则灌输）**：
```
"你必须遵守这些规则：浮力实验 ρ物 > ρ液 → 下沉..."
→ LLM 机械套用，无法处理新问题
```

✅ **新模式（知识引导 + 自主推理）**：
```
"这是各学科的知识库。请先理解原理，再自主设计实验..."
→ LLM 真正理解，自主推理，灵活应对
```

### S - Study（学习理解）

**在设计实验前，必须先完成以下思考：**

```

3. 历史案例或经典实验？

```

**完成概念理解后，分析以下问题：**


## 通用科学原则

### 1. 实验设计原则

| 原则 | 说明 |
|------|------|
| 可操作性 | 学生可在给定条件下完成 |
| 可观测性 | 结果可观察或测量 |
| 可重复性 | 相同条件下结果一致 |
| 安全性 | 无危险或可控风险 |
| 简洁性 | 聚焦核心概念 |

### 2. 变量控制原则

- **单一变量**：一次只改变一个因素
- **对照实验**：设置实验组和对照组
- **重复测量**：多次测量取平均值

### 3. 数据记录原则

- 使用标准单位
- 记录有效数字
- 注明测量条件
- 保留原始数据

## 错误预防框架

### 常见错误类型

| 类型 | 描述 | 预防策略 |
|------|------|---------|
| 操作错误 | 仪器使用不当 | 图示说明 |
| 读数错误 | 视差、估读不准 | 规范读数方法 |
| 概念误解 | 原理理解偏差 | 追问引导 |
| 计算错误 | 公式/单位误用 | 步骤示范 |
| 假设错误 | 前提条件忽略 | 明确条件 |

## 历史案例融合

### 为什么融入历史？

1. **激发兴趣**：科学家的故事引人入胜
2. **理解过程**：知道知识如何被发现
3. **培养思维**：学习科学方法
4. **情感价值**：感受科学精神

### 融合方式

```
在「核心问题」中引用：
"200多年前，伽利略在比萨斜塔...
 这个问题至今仍是物理学入门的经典问题：
 为什么轻重不同的物体下落一样快？"

在「讨论点」中引用：
"法拉第花了10年才成功...
 如果你是法拉第，遇到困难时会怎么做？"
```

## 质量检查清单

在输出前，必须自检：

- [ ] 是否先理解了概念本质？
- [ ] 变量设计是否合理？
- [ ] 是否融合了历史案例或生活应用？
- [ ] 常见错误是否覆盖操作/概念/计算？
- [ ] 实验是否可操作、可重复？
- [ ] 讨论点是否能引发深度思考？

## 优秀案例参考

### 案例结构示范

```
实验：种子萌发条件探究
学科：生物 | 年级：初一

【理解阶段】
概念：种子萌发需要什么条件？
历史：孟德尔豌豆实验（控制变量法起源）
应用：农业生产中的播种时机

【设计阶段】
核心问题：为什么农民伯伯要在春天播种？
变量：
- 自变量：温度、水分、空气
- 因变量：种子是否萌发
- 控制：种子品种、数量

【讨论点】
- 如果在夏天播种会怎样？
- 为什么有些种子可以在秋天播种？
```

**核心原则**：让 LLM 先理解，再设计。不要灌输规则，而是提供知识让它自主推理！
